Objectives

  • Students will use document.querySelectorAll to efficiently access all DOM nodes with the same id/class in their JavaScript.
  • Students will declare and manipulate data structures such as objects and arrays.
  • Students will use .forEach() to iterate through an array.
  • Students will use .addEventListener() to add click events to their HTML elements.

Key vocabulary: DOM, Selector, Event, Array, Object, Index, Conditional Statement

Demo

  1. Present the final version of the site.
    • make sure that you’re presenting your whole screen, otherwise the popups may not be visible to students.
  2. Take the quiz genuinely, clicking one answer per question. Show students that the box selected gets highlighted. NOTE: Do not change answers along the way. Each click gets recorded.
  3. After selecting the last question, show students how a result appears at the bottom of the page that was not there before.
  4. Ask students: “Why do people love these quizzes?”. Consider asking questions like “What should happen when we picked some different answers?”
    • Try to get the students to explain in their own words that the results are personalized. Reinforce that idea with the idea that the code needs to remember all the answers to come up with the personalized result.
    • Don’t spend more than a few minutes on this though.

  1. Since there is not much the students code in the final if statement, take a moment to explain how the results are being generated. The user puts themselves more in a quadrant of 2 axis through each question they answer. One axis is the “Introvert vs Extrovert” axis and the other is the “Logical vs Creative” axis. Each question moves them over one unit in one of these 2 axis. If you line up the axis as a coordinate plane, the user ends up in 1 of 4 quadrants, hence the 4 possible results: logical-introvert, logical-extrovert, creative-introvert, and creative-extrovert.

Extensions for Independent Practice

Mild

  • Add an extra question! Pick either “Introvert vs Extrovert” or “Logical vs Creative” and personalize it with your own pictures.
  • Right now, there are no pictures for either of the 4 results. Add a photo for all the results and change any other pictures or result titles you don’t like.

Medium

  • Only show one question at a time. As each question is answered, reveal the next one.
  • Add a “Take Again” button at the end of the page that is revealed along with the quiz results. When clicked, it should do the following:
    • Clear the choices (turn them white again), and
    • Hide everything but the first question

Spicy

  • Right now, the images can be selected on and off, and multiple images can be selected at the same time which both don’t make much sense. Add to the code a way to allow users to change their mind about a question. Here is what that may look like.
    • Only change the background color of the image to yellow when it is clicked, unless the user selects another option. You may have to remove the .toggle() function and use .remove() and .add() instead.
    • “Decrement” (or --) the trait that was clicked before, and increment the new trait.
    • Lock all the buttons after the last question is selected and the result is unhidden. Once all questions are selected, the user shouldn’t be able to change any of their answers.
    • Wrap the instructions in an if statement that checks if all questions have been selected or not. Once all questions have been answered, the click effect should disable. No questions should be able to be switched and the results shouldn’t change.
