Objectives

  • Use document.querySelectorAll to efficiently access all DOM nodes with the same id/class in their JavaScript.
  • Declare and manipulate data structures such as objects and arrays.
  • Use .forEach() to iterate through an array.
  • Use .addEventListener() to add click events to their HTML elements.

Key vocabulary: DOM, Selector, Event, Array, Object, Index, Conditional Statement

What We’re Building

We’re going to be making a personality quiz. We will be accessing DOM nodes and using event listeners to have the user’s quiz answers light up. Also, we will use objects and arrays within objects to store user input and display the final result.

  1. What’s happening to decide the user’s final outcome? The user is moving themselves over in a quadrant of 2 axis as they answer each question. One axis is the “Introvert vs Extrovert” axis and the other is the “Logical vs Creative” axis. Each question moves them over one unit in one of these 2 axis. If you line up the axis as a coordinate plane, the user ends up in 1 of 4 quadrants, hence the 4 possible results: logical-introvert, logical-extrovert, creative-introvert, and creative-extrovert.

Stretch Goal

Mild

  • Add an extra question! Pick either “Introvert vs Extrovert” or “Logical vs Creative” and personalize it with your own pictures.
  • Right now, there are no pictures for either of the 4 results. Add a photo for all the results and change any other pictures or result titles you don’t like.

Medium

  • Only show one question at a time. As each question is answered, reveal the next one.
  • Add a “Take Again” button at the end of the page that is revealed along with the quiz results. When clicked, it should do the following:
    • Clear the choices (turn them white again), and
    • Hide everything but the first question

Spicy

  • Right now, the images can be selected on and off, and multiple images can be selected at the same time which both don’t make much sense. Add to the code a way to allow users to change their mind about a question. Here is what that may look like.
    • Only change the background color of the image to yellow when it is clicked, unless the user selects another option. You may have to remove the .toggle() function and use .remove() and .add() instead.
    • “Decrement” (or --) the trait that was clicked before, and increment the new trait.
    • Lock all the buttons after the last question is selected and the result is unhidden. Once all questions are selected, the user shouldn’t be able to change any of their answers.
    • Wrap the instructions in an if statement that checks if all questions have been selected or not. Once all questions have been answered, the click effect should disable. No questions should be able to be switched and the results shouldn’t change.
