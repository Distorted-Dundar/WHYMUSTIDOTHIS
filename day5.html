Mini-Project 1: HTML site (With Bulma and JavaScript)

Objectives

  • Students will ideate and work in collaborative groups.
  • Students will synthesize the content they learned in days 1-4 of this course.

What We’re Building

Students will be building a webpage for a business, store, or service they’d like to start.

Share some exemplars with students, to give them a idea of the scope of project they should be aiming to build.

Exemplar A - Pineapple Fried Rice

For their subscribed customers, Matt W. offers a recipe of the week. This week we look at Pineapple Fried Rice.

Matt uses HTML with Bulma, a bit of CSS, and JS. In the JS, Matt uses a querySelector over the serving size, a variable for the “serving factor”, and innerHTML to set the new measurements of his recipe.

Exemplar B - Solar System Tour

Take a tour through the solar system. Matt K. has set up this online teaser for his stellar, interstellar tour through the solar system. You can see a bit of what Matt has to offer, but to see more, be sure to book a guided tour.

Matt uses vanilla HTML with Bulma and CSS with url backgrounds to create this website. Matt uses href linked to the id of different sections on the page to autoscroll.

Exemplar C - Crashlands Wiki

In this webpage, Kristoffer presents the game of the month, Crashlands, featured on their game selling store. Crashlands is an adventure-crafting RPG driven by an epic story arc. Get your copy today!

Kristoffer uses HTML with Bulma, and some JS: eventListeners (mouse over, and mouse out) & querySelectors

Exemplar D - My Journal

Ray’s free “My Journal” app allows users to log their daily activities all in one place. It’s simple, it’s minimal, and there are no distractions so you can stay on task.

Ray uses HTML & CSS for the basis of his page. They use JS to handcraft an input element using querySelector, eventListener (keyup) for the enter button, innerHTML to add on to the list, and a compound conditional if statement to validate the input.

The exact nature of the business is totally up to the students, but the more guidance you give them, the less likely they are to face writer’s block - here are some starter prompts, but you’re welcome to give others or narrow these down.

  • Something silly, like a company that rents umbrellas for dogs.
  • A restaurant that blends your favorite cuisine with your partner’s, like a Sushi & Pasta palace.
  • Something serious, that addresses a real need you’ve seen in your life or in your community.

Project Mode Tips

Students should work together to an HTML page for a company they will co-found with their partner - it is advisable to follow the protocols set forth in the Project Mode guide with respect to ideation, but skip the more drawn out group formation pieces. Simply assign groups randomly and send students on their way.

Since this will be the first time students are working on their own, it is important that students are aware of core repl.it features that make collaboration and group programming easier. Some of these are skills that students have already started using, but the following serves as a checklist of handy skills that are useful to demo for students. Make sure that students:

  1. Know how to invite a peer to edit. Highlight that when using repl.it, it’s not enough to copy the link from the URL. Copying the URL will only give students read access, but no write access. Instead, make sure that students know to click the share button in the upper right corner for the proper invite link to invite others to code in the same space.
  2. Know how to start threads / add comments on certain pieces of code in the repository. To do this, students simply need to right click on a specific line of code and click “Start thread”
  3. Know about the chat feature. In the bottom right corner, there’s a chat icon. Clicking on it will open up a chat where everyone viewing the project can interact with each other.
  4. Know about “forking” and how to effectively use version control. To fork, students will use the version control button on the right side column (under the File icon). There students will be able to create branches and make changes to those branches. After making a change, in the same version control panel, students will be able to pull in changes to the their main branch. Explain how this is an effective way to save a working version of their code and how to add more features without risking that version.
  5. Know to test and run their code frequently. This is not just because it’s a good practice but also because if a student’s internet disconnects changes aren’t saved until the window is refreshed. While repl.it auto saves, it only does so when one is connected to the internet. Testing the code helps students realize if they’re currently disconnected.

More information on “multiplayer” mode repl.it here

Note: Students will not likely use all topics covered so far in their projects - some may not even use JavaScript at all. That’s okay! The goal of this project day is for students to feel proud of what they’ve built, and to deepen friendships with their project partners. There will be plenty of opportunities to dig deeper into JavaScript in the coming weeks.

Notes on Friday

Fridays, especially the first and second Fridays of the course, are intentionally designed with no new content. These “flex days” are critical for student success - the extra time spent without the addition of new material allows them to deepen their understanding by shifting from cognitive to autonomous performance.

This mini-project sprint that allows them to brainstorm ideas and then try to implement them is one of the best ways to offer a comprehensive review of the topics covered earlier in the week.

That said, if it has become apparent that a specific concept proved more of a stumbling block than was anticipated, that concept is critical to know moving forward, and a large subset of students are experiencing difficulty with that topic, then if an instructor wants to offer a review session for that subset of students which covers these topics in a little more detail and formality, the team should try to adjust to make that happen.

OPTIONAL: Bulma Templates

Some students may be really excited to build out some websites with Bulma components straight from the source documentation or even from scratch, but others may be feeling a little overwhelmed. If you have some students who have expressed some trepidation around building something from scratch, you might encourage them to start with a template. This GitHub repo and Awesome Open Source both have a few great Bulma templates to start out with, and HTML5 UP also has some really beautiful templates that are made entirely from vanilla HTML and CSS - some use JavaScript though, so be careful about assuming these will be easy for students.

In general, students take to the concept of templates pretty easily, since all the code-alongs we’ve done so far have been more or less examples of “templates” which have a chunk of starter code that serve as a jumping off point for creativity. They’ll likely have questions about how to customize them, so be prepared to troubleshoot.
